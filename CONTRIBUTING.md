# Contributing to OpenShard

Thanks for your interest in contributing.

## What we'd love help with

- Routing policies and scoring logic
- Repo analyzers for new stacks
- Model profiles and capability data
- Evaluation datasets
- Provider integrations
- CLI UX improvements

## How to contribute

1. Fork the repo
2. Create a branch for your change
3. Make your changes
4. Open a pull request with a clear description of what you've done and why

## Guidelines

- Keep changes focused - one thing per PR
- Match the existing code style
- If you're adding something new, add a brief note in the PR description 
  explaining the reasoning

## Questions

Open an issue and we'll respond as soon as we can.
